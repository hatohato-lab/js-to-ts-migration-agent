# 説明書 — js-to-ts-migration-agent

この文書は、ITやAIに詳しくない方に向けて、このリポジトリ（プログラム置き場）の内容を説明するものです。
技術的な手順は README.md に、設計の背景は design/design.md にあります。

## 一言でいうと

古い形式のプログラム（JavaScript）を新しい形式（TypeScript）へ書き換えるAIと、
その書き換えが正しかったかどうかを自動で採点する仕組みを、ひとまとめにしたものです。

## 何の役に立つのか

プログラムの書き換えでは、「書き換えたつもりが、実は動きが変わっていた」という失敗が起こりがちです。
人が目で見て確認するのは手間がかかり、見落としも出ます。

このリポジトリでは、その確認を人ではなく機械に任せます。
書き換えの前と後のプログラムを両方動かし、結果が同じかどうかを自動で照合します。

たとえるなら、翻訳前の文章と翻訳後の文章に同じ質問を投げ、答えが同じになるかを比べるようなものです。
答えが一つでも違えば、その書き換えは不合格と判定されます。

また、「先に合格条件を決めてから作り、機械で採点する」という進め方（評価駆動開発）の実例にもなっています。
AIに作業を任せるとき、その結果を客観的に信頼するための基本の型です。

## 正しさをどう確かめているか

採点は次の2段階です。

1. 書き換え後のプログラムが、文法の厳しい自動検査（コンパイル）を通ること。
2. 実際に動かしたときの画面出力が、書き換え前のプログラムの出力と一致すること。

正解を人が手で用意するのではなく、書き換え前のプログラム自身を動かして正解を作る点が特徴です。
人の思い込みが正解に混ざらないため、誰がいつ実行しても同じ判定になります。

さらに、採点プログラム自体が壊れていないかも点検します。
わざと間違えた書き換えを採点させ、きちんと不合格になることを確認する自己検査を備えています。
採点者が正しく減点できるかを、事前に試験しておくという考え方です。

なお、この確認は用意した練習問題での出力一致であり、あらゆる場合の完全な同一を証明するものではありません。

## フォルダとファイルの案内

- README.md … 使い方の手順書（技術者向け）。
- 説明書.md … この文書。
- .claude/agents/ … 書き換えを担当するAIへの指示書。
- eval/oracle.py … 採点プログラムの本体。
- eval/corpus/ … 採点用の練習問題集。小さなプログラムが3つ入っています。
- design/design.md … なぜこの設計にしたかの記録。
- .github/workflows/ … 更新のたびに採点の自己検査を自動実行する設定。
- tsconfig.json ほか … 開発用の設定ファイル。

## 用語の小辞典

- JavaScript … Webページなどで広く使われているプログラミング言語。
- TypeScript … JavaScriptに「型」という誤り防止の仕組みを加えた言語。
- 移行（マイグレーション） … プログラムを別の言語や新しい形式へ書き換えること。
- オラクル … 合格・不合格を機械的に判定する採点役。ここでは採点プログラムを指します。
- エージェント … 指示書に従って作業を進めるAIのこと。
